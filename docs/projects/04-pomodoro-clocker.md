# Pomodoro Clocker

## Product Vision

Ми створюємо вебдодаток, який допомагає користувачу працювати сфокусовано за помодорною системою: запускати робочі сесії, робити перерви та бачити власну статистику продуктивності.

Продукт має бути легким у використанні: користувач відкриває застосунок, налаштовує цикл і одразу починає роботу.

## Для кого продукт

- Студент, який хоче краще організувати навчання.
- Розробник або спеціаліст, який працює блоками фокусованого часу.
- Користувач, який хоче бачити статистику своєї продуктивності.

## Проблема користувача

Без таймера легко втратити фокус або працювати без перерв. Користувачу потрібен інструмент, який веде його через цикл роботи й відпочинку та зберігає історію виконаних сесій.

## Product Goal

До кінця проєкту команда має показати працюючий таймер із налаштуваннями, автоматичними переходами між роботою і перервою та статистикою завершених помодорів.

## MVP Scope

### Must Have

- Реєстрація та вхід користувача.
- Таймер робочої сесії.
- Коротка перерва.
- Довга перерва.
- Налаштування тривалості роботи та перерв.
- Налаштування кількості помодорів до довгої перерви.
- Запуск, пауза, продовження та скидання таймера.
- Автоматичне перемикання між етапами.
- Збереження завершених помодорів.
- Статистика за день і тиждень.

### Should Have

- Візуальне відображення поточного етапу.
- Звукове або browser notification сповіщення після завершення етапу.
- Історія сесій.
- Порожній стан, коли статистики ще немає.
- Збереження персональних налаштувань.

### Could Have

- Прив'язка помодора до задачі.
- Теги для сесій.
- Графіки продуктивності.
- Темна тема.
- Найпродуктивніший день.

## User Stories

- Як користувач, я хочу налаштувати тривалість помодора, щоб працювати у зручному для мене ритмі.
- Як користувач, я хочу запускати й ставити таймер на паузу, щоб контролювати робочу сесію.
- Як користувач, я хочу отримувати сигнал після завершення етапу, щоб не стежити за часом вручну.
- Як користувач, я хочу бачити кількість завершених помодорів за день, щоб оцінити продуктивність.
- Як користувач, я хочу бачити історію сесій, щоб аналізувати свій режим роботи.

## Основні екрани

- Таймер.
- Налаштування таймера.
- Статистика.
- Історія сесій.
- Профіль користувача.

## Acceptance Criteria

- Користувач може запустити повний цикл: робота, коротка перерва, робота.
- Після заданої кількості помодорів система переходить до довгої перерви.
- Пауза зупиняє таймер, а продовження відновлює його з того самого часу.
- Завершений помодор зберігається в історії.
- Статистика за день показує правильну кількість завершених сесій.
- Налаштування користувача зберігаються після перезавантаження сторінки або повторного входу.

## Out of Scope для MVP

- Desktop system tray.
- Мобільний застосунок.
- Командна статистика.
- Інтеграція з календарем.
- Складні productivity reports.

## Definition of Done

- Усі Must Have сценарії працюють.
- Таймер стабільно поводиться при паузі, скиданні та завершенні етапу.
- Є тестовий користувач із прикладами історії.
- Є README з інструкцією запуску.
- Команда може показати демо таймера та статистики.

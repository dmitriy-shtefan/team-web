# Вебдодаток для обліку персональних фінансів

## Product Vision

Ми створюємо застосунок, який допомагає користувачу бачити, куди йдуть гроші, скільки він отримує, скільки витрачає і який баланс має на різних гаманцях. Продукт має бути не бухгалтерською системою, а простим персональним фінансовим помічником.

Головна цінність: користувач швидко додає операції та отримує зрозумілу картину своїх фінансів.

## Для кого продукт

- Користувач, який хоче контролювати особисті витрати.
- Студент або працівник, який веде кілька джерел доходу.
- Людина, яка користується кількома гаманцями або валютами.

## Проблема користувача

Без системного обліку користувач не розуміє, скільки витрачає за категоріями, який має реальний баланс і де можна оптимізувати витрати. Таблиці незручні для щоденного використання, а банківські застосунки не завжди дають повну картину.

## Product Goal

До кінця проєкту команда має показати застосунок, у якому користувач додає доходи й витрати, бачить баланс гаманців і переглядає графіки за період.

## MVP Scope

### Must Have

- Реєстрація та вхід користувача.
- Створення гаманців.
- Вибір валюти для гаманця.
- Створення категорій витрат.
- Створення джерел доходу.
- Додавання доходу.
- Додавання витрати.
- Редагування та видалення операції.
- Перегляд списку операцій.
- Фільтрація за датою, гаманцем і категорією.
- Розрахунок балансу по гаманцю.
- Графік доходів і витрат за період.

### Should Have

- Dashboard із загальним балансом.
- Порожні стани для нових користувачів.
- Валідація сум і дат.
- Розділення даних між користувачами.
- Базовий звіт по категоріях витрат.

### Could Have

- Місячні бюджети.
- Регулярні операції.
- Імпорт або експорт CSV.
- Конвертація валют.
- Попередження про перевищення бюджету.

## User Stories

- Як користувач, я хочу створити гаманець, щоб вести облік окремо для картки, готівки або рахунку.
- Як користувач, я хочу додати витрату з категорією, щоб потім бачити структуру витрат.
- Як користувач, я хочу додати дохід, щоб баланс був актуальним.
- Як користувач, я хочу фільтрувати операції за періодом, щоб аналізувати конкретний місяць.
- Як користувач, я хочу бачити графік витрат, щоб швидко зрозуміти фінансову ситуацію.

## Основні екрани

- Dashboard.
- Список операцій.
- Додавання або редагування операції.
- Гаманці.
- Категорії.
- Звіти.
- Налаштування валют.

## Acceptance Criteria

- Користувач може створити гаманець і додати до нього дохід.
- Користувач може додати витрату, і баланс гаманця зменшується.
- Після редагування або видалення операції баланс перераховується.
- Фільтр за періодом показує тільки операції з вибраного діапазону дат.
- Графік доходів і витрат відповідає даним у списку операцій.
- Користувач не бачить фінансові дані інших користувачів.

## Out of Scope для MVP

- Банківські інтеграції.
- Реальні курси валют із зовнішніх API.
- Податковий облік.
- Спільні сімейні бюджети.
- Сканування чеків.

## Definition of Done

- Усі Must Have сценарії працюють.
- Є тестовий користувач із прикладами доходів і витрат.
- Баланси та графіки рахуються коректно на тестових даних.
- Є README з інструкцією запуску.
- Команда може показати сценарій додавання операцій і аналізу витрат.

# Вебдодаток "Список справ"

## Product Vision

Ми створюємо простий task management застосунок для особистих справ. Користувач має швидко записати задачу, визначити пріоритет і дедлайн, відстежити виконання та не загубити завершені задачі.

Продукт має бути легким, швидким і зрозумілим без навчання.

## Для кого продукт

- Студент, який планує навчальні завдання.
- Користувач, який веде особистий список справ.
- Невелика команда, якщо команда вирішить додати спільні списки як додаткову можливість.

## Проблема користувача

Задачі легко губляться в нотатках або чатах. Користувачу потрібне місце, де видно активні справи, дедлайни, пріоритети та виконані задачі.

## Product Goal

До кінця проєкту команда має показати застосунок, у якому користувач створює списки й задачі, фільтрує їх за статусом, задає дедлайни та бачить виконані задачі в архіві.

## MVP Scope

### Must Have

- Реєстрація та вхід користувача.
- Створення списку справ.
- Створення задачі.
- Редагування задачі.
- Видалення задачі.
- Позначення задачі як виконаної.
- Категорії задач.
- Пріоритет задачі.
- Дедлайн задачі.
- Фільтр за статусом: усі, активні, виконані.
- Сортування за дедлайном або пріоритетом.
- Архів виконаних задач.

### Should Have

- Dashboard зі списками.
- Порожні стани.
- Валідація назви задачі та дедлайну.
- Швидка зміна статусу задачі.
- Лічильники активних і виконаних задач.

### Could Have

- Нагадування.
- Повторювані задачі.
- Підзадачі.
- Drag and drop.
- Kanban view.
- Спільні списки для кількох користувачів.

## User Stories

- Як користувач, я хочу створити список справ, щоб розділяти задачі за контекстом.
- Як користувач, я хочу додати задачу з дедлайном, щоб не забути виконати її вчасно.
- Як користувач, я хочу встановити пріоритет, щоб розуміти, що важливіше.
- Як користувач, я хочу позначити задачу виконаною, щоб бачити прогрес.
- Як користувач, я хочу фільтрувати задачі за статусом, щоб не змішувати активні й завершені справи.
- Як користувач, я хочу переглядати архів, щоб знайти раніше виконані задачі.

## Основні екрани

- Dashboard.
- Список задач.
- Форма задачі.
- Категорії.
- Архів.
- Профіль користувача.

## Acceptance Criteria

- Користувач може створити список і додати до нього задачу.
- Задачу можна відредагувати, видалити та позначити виконаною.
- Фільтр активних задач не показує виконані задачі.
- Сортування за дедлайном показує найближчі дедлайни першими.
- Виконана задача з'являється в архіві.
- Користувач бачить тільки власні списки й задачі.

## Out of Scope для MVP

- Повноцінний project management.
- Інтеграція з календарем.
- Push notifications.
- Командні дошки з ролями.
- Time tracking.

## Definition of Done

- Усі Must Have сценарії працюють.
- Є тестовий користувач із кількома списками й задачами.
- Є README з інструкцією запуску.
- Є опис основних сценаріїв користувача.
- Команда може показати сценарій: створення задачі, фільтрація, виконання, архів.
